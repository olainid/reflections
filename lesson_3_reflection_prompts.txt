When would you want to use a remote repository rather than keeping all your work
local?

    When working with more people in the same project

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

    To have absolute control of the changes

Describe the differences between forks, clones, and branches.  When would you
use one instead of another?

    Forks: copy someones repository in my GitHub
    Clone: copy from GitHub (or maybe local) in my computer
    Branches: Other version of my code

stored locally?

    That you can make the changes that you want from the most advanced version without risking to destroy the remote code
